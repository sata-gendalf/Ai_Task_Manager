# AI Task Manager

Приложение для управления задачами с автоматическим определением приоритета и категории.

## Описание

Система анализирует текст задачи (например "Подготовить презентацию к пятнице") и определяет:
- Приоритет (низкий, средний, высокий)
- Категорию (работа, учёба, личное)

Возможности:
- Регистрация и авторизация
- Приватные задачи (видны только владельцу)
- Управление статусом задачи
- Удаление задач

## Технологии

- React + Vite (фронтенд)
- Node.js + Express (бэкенд)
- PostgreSQL (БД)

## Установка и запуск

### Что нужно

1. Node.js v16+
2. PostgreSQL 12+
3. Python 3.8+

### Установка БД

```bash
psql -U postgres
CREATE DATABASE ai_task_manager;
\c ai_task_manager
\i database/schema.sql
```

### Бэкенд

```bash
cd backend
npm install
cp .env.example .env
# Отредактируйте .env с необходимыми данными
npm run dev
```

### Python сервис (опционально)

```bash
cd python-service
python -m venv venv
source venv/bin/activate  # или venv\Scripts\activate на Windows
pip install -r requirements.txt
python main.py
```

### Фронтенд

```bash
cd frontend
npm install
npm run dev
```

После завершения всех шагов приложение доступно на http://localhost:5173

## Использование

1. **Регистрация** - необходимо указать email и пароль
2. **Вход в аккаунт** - авторизация через созданный аккаунт
3. **Создание задачи** - ввод текста в поле и отправка на анализ
4. **Автоматический анализ** - система определяет приоритет и категорию
5. **Управление задачами** - просмотр, обновление статуса или удаление задач

---

## API Endpoints

Основные endpoints:

```
POST /auth/register       # Регистрация
POST /auth/login          # Вход

GET /tasks                # Получить задачи
POST /tasks               # Создать задачу
PUT /tasks/:id           # Обновить задачу
DELETE /tasks/:id        # Удалить задачу
```

Все запросы (кроме регистрации и входа) требуют заголовок: `Authorization: Bearer <токен>`

## Решение проблем

### Ошибка подключения к БД
- Проверьте что PostgreSQL запущен
- Убедитесь в правильности данных в `.env`
- Убедитесь что БД существует

### Python сервис не работает
- Это нормально, задачи получают значения по умолчанию
- Для полного функционала запустите сервис отдельно

### Ошибки авторизации
- Проверьте формат email
- Пароль должен содержать минимум 6 символов
- Email должен быть уникальным в системе

### Задачи не сохраняются
- Просмотрите логи бэкенда
- Откройте консоль браузера (F12) для деталей ошибки
- Перезагрузите страницу и повторите попытку

## Принцип работы

1. Пользователь вводит задачу
2. Отправляется на бэкенд
3. Проверяется JWT токен
4. Отправляется в Python сервис для анализа
5. Python возвращает приоритет и категорию
6. Всё сохраняется в PostgreSQL
7. Фронтенд обновляет список
