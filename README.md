# Task Manager

Приложение для управления задачами с автоматическим определением приоритета и категории по тексту.

## Описание

Система анализирует текст задачи (например «Подготовить презентацию к пятнице») и определяет:
- **Приоритет:** низкий, средний, высокий
- **Категорию:** работа, учёба, личное, здоровье, общее

Возможности:
- Регистрация и авторизация
- Приватные задачи (видны только владельцу)
- Управление статусом задачи
- Удаление задач

## Установка и запуск

### Что нужно

1. Node.js v16+
2. PostgreSQL 12+
3. Python 3.8+

### Порты сервисов

| Сервис     | Порт | Команда запуска              |
|------------|------|------------------------------|
| PostgreSQL | 5432 | —                            |
| Backend    | 5000 | `npm run dev` в `backend/`   |
| Python     | 8000 | `python main.py` в `python-service/` |
| Frontend   | 5173 | `npm run dev` в `frontend/`  |

Для работы приложения нужно запустить **все три сервиса** (backend, python-service, frontend) и иметь доступную базу PostgreSQL.

### Установка БД

```bash
psql -U postgres
CREATE DATABASE ai_task_manager;
\c ai_task_manager
\i database/schema.sql
```

### Бэкенд

```bash
cd backend
npm install
cp .env.example .env
# Отредактируйте backend/.env — данные БД, JWT_SECRET, PYTHON_SERVICE_URL
npm run dev
```

### Python сервис

Сервис анализа текста задач. Запускайте его вместе с backend — без него задачи создаются с приоритетом `medium` и категорией `general`.

```bash
cd python-service
python -m venv venv
source venv/bin/activate  # или venv\Scripts\activate на Windows
pip install -r requirements.txt
python main.py
```

### Фронтенд

```bash
cd frontend
npm install
npm run dev
```

После запуска всех сервисов приложение доступно на http://localhost:5173

## Использование

1. **Регистрация** — укажите email и пароль (не менее 6 символов)
2. **Вход** — авторизация через созданный аккаунт
3. **Создание задачи** — введите текст и отправьте форму
4. **Автоматический анализ** — система определяет приоритет и категорию
5. **Управление задачами** — просмотр, смена статуса или удаление

## API Endpoints

```
POST /api/auth/register    # Регистрация
POST /api/auth/login       # Вход
GET  /api/auth/me          # Текущий пользователь (восстановление сессии)

GET    /api/tasks          # Получить задачи
POST   /api/tasks          # Создать задачу
PUT    /api/tasks/:id      # Обновить задачу
DELETE /api/tasks/:id      # Удалить задачу
```

Все запросы, кроме регистрации и входа, требуют заголовок: `Authorization: Bearer <токен>`

## Решение проблем

### Ошибка подключения к БД
- Проверьте, что PostgreSQL запущен
- Убедитесь в правильности данных в `backend/.env`
- Убедитесь, что БД `ai_task_manager` создана и схема применена

### Python сервис не работает
- Убедитесь, что сервис запущен на порту 8000
- Проверьте переменную `PYTHON_SERVICE_URL` в `backend/.env`
- Если сервис недоступен, backend сохранит задачу с приоритетом `medium` и категорией `general`

### Ошибки авторизации
- Проверьте формат email
- Пароль должен содержать минимум 6 символов
- Email должен быть уникальным в системе

### Задачи не сохраняются
- Просмотрите логи бэкенда
- Откройте консоль браузера (F12) для деталей ошибки
- Перезагрузите страницу и повторите попытку

## Принцип работы

1. Пользователь вводит задачу
2. Текст отправляется на бэкенд
3. Проверяется JWT токен
4. Текст передаётся в Python сервис для анализа
5. Python возвращает приоритет и категорию
6. Данные сохраняются в PostgreSQL
7. Фронтенд обновляет список задач
